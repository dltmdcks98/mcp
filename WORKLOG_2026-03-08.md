# MCP Worklog (2026-03-08)

## 핵심 요약
- `gemini-mcp` 프로젝트를 기준으로 Gemini CLI 연동 Streamable HTTP MCP 서버 구성을 정리함.
- 서버 엔드포인트(`/mcp`, `/healthz`)와 세션 기반 요청 처리 구조를 정비함.
- OpenClaw 등 외부 클라이언트에서 사용할 수 있도록 실행 방법과 연동 포인트를 문서화함.

## 구현/구성 내역
- 런타임: Node.js ESM 기반(`server.mjs`)
- 의존성: `@modelcontextprotocol/sdk`, `express`
- MCP Tool: `gemini.generate`
  - 입력: `prompt`(필수), `model`(선택), `output_format`(선택: `text|json`)
- 기본 동작:
  - Gemini CLI 실행 시 `--model`, `--prompt`, `-y` 옵션 사용
  - `output_format=json` 선택 시 JSON 파싱 결과를 MCP `json` 컨텐츠로 반환
  - 실패 시 `isError: true` 형식으로 에러 응답

## 운영 포인트
- 기본 주소: `http://127.0.0.1:8765/mcp`
- 헬스체크: `http://127.0.0.1:8765/healthz`
- 주요 환경변수: `PORT`, `HOST`, `GEMINI_CLI_BIN`
